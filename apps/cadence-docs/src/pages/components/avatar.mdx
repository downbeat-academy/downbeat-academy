# Avatar

The `Avatar` component is used to dislay an image of a person or user. It accepts either:

-   `imageUrl`: a URL from more or less any source.
-   `imageObject`: an image object which supports framework-specific images like `next/image`.

If neither of these are present a branded placeholder will be rendered instead.

    <Avatar
        name='Jory Tindall'
    />
```

    <Avatar
        name='Jory Tindall'
        imageObject={avatarImage}
    />
```

## Size

The `Avatar` component supports three sizes, `small`, `medium` (default), and `large`.

```
    <Avatar name="Jory Tindall" imageUrl="/jory-tindall-avatar.jpg" size="small" />
    <Avatar name="Jory Tindall" imageUrl="/jory-tindall-avatar.jpg" size="medium" />
    <Avatar name="Jory Tindall" imageUrl="/jory-tindall-avatar.jpg" size="large" />
```

<Avatar name="Jory Tindall" imageUrl="/jory-tindall-avatar.jpg" size="medium" />
<Avatar name="Jory Tindall" imageUrl="/jory-tindall-avatar.jpg" size="large" />

## Border

By default, the `Avatar` component isn't wrapped in a border, but one can be added via the boolean `hasBorder` property if more differentiation from a dark background is necessary.

```
    <Avatar name="Jory Tindall" imageUrl="/jory-tindall-avatar.jpg" hasBorder={true} />
```

<Avatar
	name="Jory Tindall"
	imageUrl="/jory-tindall-avatar.jpg"
	hasBorder={true}
/>

## Rounded

By default the container of the `Avatar` component enforces a fully-rounded border radius, but the component also supports a square layout with the `rounded` boolean property.

```
    <Avatar
        name='Jory Tindall'
        imageUrl="/jory-tindall-avatar.jpg"
        rounded={false}
    />
```

<Avatar
	name="Jory Tindall"
	imageUrl="/jory-tindall-avatar.jpg"
	rounded={false}
/>

## Name

The `name` property doesn't have any effect on the visual of the component except in the case of the placeholder, which generates a unique SVG pattern based on the name string. Behind the scenes, the name is the alt text for the image and should be used as such for accessibility.

## Avatar Group

Multiple `Avatar` components can be grouped together with the `AvatarGroup` component which acts as a wrapper for the nested components. It doesn't accept an array of `Avatars`, rather an object array of `Avatars` to reduce the nested dependencies.

The `AvatarGroup` accepts several properties to determine the layout of the nested components:

-   `overlap`: a boolean property that deteremines whether the nested components overlap slightly
-   `overlapSpacing`: determines how much the nested components overlap
-   `direction`: renders the nested components either vertically or horizontally.
-   `isInteractive`: Adds a slight animation when the nested components are interacted with/hovered. This is best paired when the `Avatar` components are wrapped in a routing mechanism.

With overlap

<AvatarGroup overlap={true} avatars={avatars} />

Overlap spacing compact

<AvatarGroup overlap={true} avatars={avatars} overlapSpacing="compact" />

Overlap spacing comfortable

<AvatarGroup overlap={true} avatars={avatars} overlapSpacing="comfortable" />

No overlap

<AvatarGroup avatars={avatars} />

Is interactive

<AvatarGroup
	overlap={true}
	avatars={avatars}
	overlapSpacing="compact"
	isInteractive={true}
/>
